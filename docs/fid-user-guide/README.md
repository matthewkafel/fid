## Section Summaries

### 1. Fiduciary Manager (pages 5–20)
The Fiduciary Manager home page is accessed via the Fiduciary link in the menu bar. It provides tabs for searching beneficiaries, fiduciaries, countries, and ZIP codes. The Admin Task section shows task queues, work queue exports, and assignment configuration. Covers:
- Tasks Queue Views (My Assigned, My Team, Fiduciary Hub)
- Customizing Tasks Queue Views
- Admin Task Actions (Transfer, Reassign, Unassign, Update Priority)
- My Work Queue Exports
- Accounting Audit Tools list view
- Field Exam Reports list view
- Benefits Threshold management
- Fiduciary Manager Search Features (Beneficiaries, Fiduciaries, Countries, ZIP Codes)

### 2. Admin Task Assignment Configuration (pages 21–25)
From the Admin Task Assignment Configuration page, authorized users can run or schedule admin task auto-assignment, and configure rules for routing tasks to teams and users within a Fiduciary Hub. Covers:
- Rule Sets and Rules configuration
- Adding, editing, and deleting Rules
- Scheduling and running auto-assignment

### 3. Beneficiary Profile (pages 26–46)
The Beneficiary Profile page is accessed from search results or fiduciary profile links. Users can manage all beneficiary profile data, establish EPs, and associate beneficiaries with fiduciaries. Covers:
- Personal Information (name, SSN, dates, ratings)
- Physical Address and Mailing Address management
- Contact Information
- Active Fiduciaries list
- Establishing EPs from the Beneficiary Profile
- EP History
- Field Exam History
- Accounting Audit Tools history
- Related Records
- Correspondence

### 4. Fiduciary Profile (pages 47–53)
The Fiduciary Profile page is accessed from search results. Users can manage all fiduciary profile data. Covers:
- Person and Organization fiduciary profiles
- Active Beneficiaries list
- Information Exam
- Organization details
- Establishing EPs from Fiduciary Profile

### 5. Admin Tasks (pages 54–57)
The Admin Task page allows users to view, create, edit, assign, and complete administrative tasks. Covers:
- Creating admin tasks
- Editing admin tasks
- Assigning and completing tasks
- Admin task detail fields

### 6. Establishing EPs (pages 58–61)
Describes the types of EPs that can be established from Fiduciary Manager and the process for doing so. Covers:
- EP590 (Initial Appointment Field Exam)
- EP290 (Fund Usage Review)
- EP400 (Eligibility Determination)

### 7. EP Overview (pages 62–81)
After EPs are established, they are managed from the EP Overview page. The page is divided into sections including Info, Notes, and type-specific sections. Covers:
- EP Info and Notes
- EP590 Initial Appointment sections (General, Appointment, Field Exam, Accounting)
- EP290 Fund Usage Review sections
- EP400 Eligibility Determination sections
- Adding and managing activities
- Suspending and closing EPs
- Setting activities and statuses

### 8. Field Exams (pages 82–94)
Field Exam Reports are used to document field exams associated with EP590 claims. Covers:
- Field Exam Report page overview
- Report sections (General, Beneficiary, Environment, etc.)
- Saving, completing, and generating field exam reports
- Non-Fiduciary Program field exam reports

### 9. Misuse Records (pages 95–107)
The Misuse Records page documents fiduciary misuse determinations. Covers:
- Misuse Record fields and sections
- Creating and editing misuse records
- Misuse Determination Memos
- Reconsideration Memos

### 10. Accounting Audit Tools (pages 108–118)
The Accounting Audit Tools page shows detailed information on accounting audits for fiduciaries. Covers:
- Accounting Audit Tool overview fields
- Accounting submission review
- Approving/disapproving accountings
- Accounting Waivers
- Inactivating Accounting Audit Tools

### 11. Audit History (page 119)
The Audit History page shows a chronological log of changes made to any record in the system. Covers:
- Beneficiary profile audit history
- Fiduciary profile audit history
- Field exam report audit history
- Accounting audit tool audit history
- Admin task audit history
- ZIP code audit history

### 12. Acronyms (page 120)
Full list of acronyms used throughout the user guide.

## Copilot / AI Readability

**All 212 UI screenshots are fully readable by GitHub Copilot and other AI assistants.**

Each screenshot in the section files is accompanied by its OCR-extracted text content inside a collapsible `<details>` block. This means every field label, button name, column header, navigation breadcrumb, and data value visible in the UI is available as searchable plain text — Copilot can answer questions like:

- *"What columns are shown in the My Assigned Tasks Queue?"*
- *"What fields are on the Beneficiary Profile page?"*
- *"What are the filter options in the Narrow Results pane?"*
- *"What buttons are available on the Field Exam Report page?"*

The OCR text was extracted using Tesseract 5.x (`--psm 6` mode) directly from the screenshot images. Note that OCR may occasionally misread characters due to UI font rendering, but the key terms (field names, section titles, button labels) are reliably captured.
